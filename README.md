#### 쇼핑몰 결제시스템 구축 테스트
주문한 상품옵션별로 주문이 생성되며, 구매자는 한번에 여러개의 주문을 넣고 한번에 결제를 진행.

#### 기능구현 테스트
1. 주문 목록을 입력받아 총 주문가격, 할인 종류별 할인 금액, 최종 결제해야할 금액을 계산하여 반환하는 기능
*** 최종 결제 금액은 구매자에게 가장 유리한(할인이 많이 되는) 정책을 적용한 결과로 반환 되어야 합니다.
2. 결제번호로 계산된 결제 내역을 조회하는 기능.
3. 결제번호로 계산된 결제 내역을 삭제하는 기능. (결제가 완료된 내역은 삭제/수정되지 않아야 함.)
4. 결제번호로 결제 완료 처리 기능 (결제가 완료되어, 삭제/수정이 불가능한 상태로 만듬)
